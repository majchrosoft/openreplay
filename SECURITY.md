# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

**DO NOT POST SECURITY ISSUES GITHUB ISSUES!**

Email us directly at: security@openreplay.example.com

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes

## Security Updates

Security patches will be released as soon as possible after verification.
We'll maintain backward compatibility for security fixes when possible.

## Security Best Practices

1. **Never commit credentials** - Use environment variables
2. **Keep dependencies updated** - Run `pip install -U -r requirements.txt`
3. **Review changes** - All PRs are reviewed before merging
4. **Test thoroughly** - All tests must pass before merging

## Security Audit

We welcome third-party security audits. Contact us to schedule one.
